# PPT制作说明书：基于MATLAB/Simulink的机器人端无线控制系统设计

## 1. 给PPT制作者的总要求

这份PPT要讲清楚一件事：本设计不是单独做电机、视频或射频，而是把机器人端的驱动控制、视频采集和无线通信放在同一个系统里，用 MATLAB R2025a/Simulink 建模，并用真实仿真结果证明方案在软件模型中是自洽的。

PPT不要堆大段文字。每页只讲一个重点，尽量用“图 + 3条解释”的方式呈现。听众是初学者，所以要先解释系统为什么这样分，再解释每个模块做什么，最后再看仿真数据说明效果。

报告中所有图都在 `output/figures/`。优先使用这些图，不要重新截图模糊图片。

## 2. 推荐PPT结构

### 第1页：标题页

标题：基于 MATLAB/Simulink 的机器人端无线控制系统设计

包含：课程名称、姓名、班级、指导老师、日期。缺失信息用“待填写”。

讲述重点：这是一个机器人端远程无线控制系统设计，不是单纯电机控制实验。

### 第2页：任务背景和设计目标

建议放图：`system_architecture.png`

讲清楚三句话：

- 机器人端需要接收控制命令，同时把视频信息传给远程端。
- 电机控制要求可靠、实时；视频链路要求带宽足够；射频链路要求距离和稳定性。
- 本设计用 MATLAB R2025a/Simulink 做系统仿真，不使用 Vitis。

### 第3页：总体系统架构

建议放图：`system_architecture.png`

讲述逻辑：

1. 摄像头采集视频。
2. DSP 负责电机闭环控制和数据调度。
3. 射频模块负责 VHF 无线收发。
4. 天线完成远距离通信。

注意：不要说 DSP 完成所有视频压缩。报告方案中视频压缩由外置 H.264 编码器承担，DSP 主要负责配置、调度和状态监测。

### 第4页：DSP选型

建议做成表格。

核心结论：选择 TI TMS320F28379D。

原因：

- 有 PWM，适合电机驱动控制。
- 有 ADC，能采集电流和母线电压。
- 有 QEP，能读取编码器速度。
- 有 SPI/SCI/CAN，能连接视频模块和射频模块。

讲述方式：不要只说“性能强”，要说它的外设正好对应任务需求。

### 第5页：驱动器控制模块

建议放图：`motor_control_circuit.png`

讲清楚信号流：

1. DSP 输出 PWM。
2. PWM 经过隔离驱动器。
3. 功率桥驱动电机。
4. 电流、电压、编码器反馈回 DSP。

重点强调：电机控制不是开环调速，而是有反馈、有保护的闭环系统。

### 第6页：电机控制算法

建议放图：`simulink_model_control_subsystem.png`

讲述重点：

- 控制器以目标转速和实际转速的误差为输入。
- PID 输出电压指令。
- 电压指令经过母线限幅和电流保护。
- 最终转成 PWM 占空比控制电机。

初学者解释：PID 可以理解为“现在差多少、过去积累差多少、变化趋势有多快”三部分共同决定控制力度。

### 第7页：R2025a专业化Simulink系统模型

建议放图：`simulink_model_top.png`

这一页要突出改进点。

讲清楚：模型不是一个简单控制环，而是分成八个子系统：

- Command_Scheduler：命令调度
- DSP_Control_Algorithm：DSP控制算法
- PWM_Gate_Driver_HBridge：PWM与隔离驱动
- DC_Motor_Electromechanical_Plant：功率级与电机
- ADC_QEP_Sensor_Acquisition：传感采集
- Video_Capture_Codec_Buffer：视频采集链路
- VHF_RF_Transceiver_Channel：射频链路
- Supervisor_Performance_Monitors：性能监测

讲述重点：这样的分层结构更接近工程系统，也便于后续扩展硬件在环测试。报告生成时记录到模型有 83 个 block、74 条连线，并通过了 0.25 s 的 Simulink 烟雾仿真。

### 第8页：Agentic Toolkit与模型审查

建议放图：`simulink_model_top.png`，旁边配一个小表格。

讲清楚三点：

- 已使用 MATLAB 官方 Agentic Toolkit、Simulink Agentic Toolkit 和 MATLAB MCP Server 做 R2025a 初始化检查。
- `output/logs/agentic_toolkit_status.json` 记录 SATK 初始化通过、MCP Server 存在、`shareMATLABSession` 可用。
- 这些工具强化的是建模流程、接口检查和可追溯记录，不代表已经做了硬件测试。

一句话解释给初学者：Agentic Toolkit 可以理解为让 AI 助手按 MathWorks 的建模规范检查和组织 Simulink 工程，但仿真数据仍然必须由 MATLAB 脚本真实运行生成。

### 第9页：视频捕获模块

建议放图：`video_capture_circuit.png`

讲清楚：

- 摄像头输出视频。
- 帧缓存解决速率和时钟差异。
- H.264 编码器降低带宽。
- DSP 监控缓存状态和码率，不直接承担全部压缩计算。

不要讲成“DSP实时压缩高清视频”，这不符合报告方案。

### 第10页：射频集成模块

建议放图：`rf_integrated_circuit.png`

讲清楚模块组成：

- DAC/ADC
- 调制解调
- 频率合成
- 滤波器组
- LNA/PA
- 天线匹配

重点：控制信号和视频数据占用不同频带。控制链路优先保证可靠性，视频链路优先保证吞吐量。

### 第11页：MATLAB/Simulink仿真流程

建议放图：`software_flow.png`

讲述流程：

1. 加载参数。
2. 运行电机控制仿真。
3. 运行视频链路仿真。
4. 运行射频链路仿真。
5. 校验结果。
6. 自动生成报告。

强调：箭头表示数据和流程方向，所有结果来自脚本运行，不是手写数据。

### 第12页：系统性能总览

建议放图：`system_performance_summary.png`

讲述重点：

- 电机响应时间、超调量反映控制性能。
- 视频载荷和缓存反映传输压力。
- SNR 反映射频链路裕量。

这一页是全局结果，不要展开太多公式。先让听众知道系统整体表现。

### 第13页：电机仿真结果

建议放图：`motor_response.png`

重点数据：

- 90% 上升时间：0.173 s
- 超调：9.55%
- 正向稳态误差：0.95 rpm
- 最终速度：约 -600 rpm

讲述逻辑：电机能快速跟踪目标转速，能完成方向反转，电流和 PWM 占空比变化可观察。

### 第14页：视频链路仿真结果

建议放图：`video_buffer.png`

重点数据：

- 平均视频载荷：36.24 Mbps
- 折算约 4.53 MB/s
- 丢帧数：0
- 最大缓存占用：4.89 MB

讲述逻辑：视频链路满足任务书“3 M以上”的带宽要求，并且缓存没有持续溢出。

### 第15页：射频链路仿真结果

建议放图：`rf_link_ber.png`

重点数据：

- 距离条件：10 km
- 最低控制信道 SNR：61.56 dB
- 最低视频信道 SNR：44.76 dB
- BER 曲线用于观察调制可靠性。

注意：这是软件链路预算，不是外场实测。PPT中必须说清楚。

### 第16页：结论

建议用三条结论：

1. 完成了机器人端电机控制、视频采集和射频链路的系统级设计。
2. 基于 MATLAB R2025a、Simulink Agentic Toolkit 和分层模型生成了可复现实验结果。
3. 仿真表明方案在给定模型参数下满足电机响应、视频带宽和射频链路预算要求。

最后一句建议：后续工作是进行硬件在环测试、真实电机测试和射频外场测试。

## 3. PPT中必须避免的错误

- 不要说已经完成真实硬件测试。
- 不要说已经完成 10 km 外场通信实验。
- 不要说 DSP 独立完成 H.264 视频压缩。
- 不要把仿真图说成示波器实测图。
- 不要删掉“MATLAB/Simulink 仿真”这个边界。
- 不要把报告中的数据手动改成更好看的数值。

## 4. 推荐视觉风格

- 每页标题控制在 12 个字左右。
- 每页正文不超过 4 行。
- 图放大，文字少写。
- 曲线图尽量直接使用 `output/figures/` 中的 PNG。
- 颜色不要过多，保持白底、深色文字、蓝/橙/绿等少量强调色。

## 5. 一句话汇报主线

本设计用 TMS320F28379D 作为机器人端 DSP 主控，通过 MATLAB R2025a/Simulink 建立电机控制、视频传输和 VHF 射频链路的分层仿真模型，并用真实生成的数据验证了系统在控制响应、视频带宽和通信链路预算方面的可行性。
